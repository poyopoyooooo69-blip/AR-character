# お料理キャラクターAR

QRコードからスマホで開き、同じQRコードを座標マーカーとして3Dキャラクターと料理オブジェクトを表示するWebAR試作です。

## できること

- iPhone・Androidのカメラ映像からQRコードの四隅を追跡
- PDFのQR実寸と四隅の遠近変形から、距離・傾き・回転を含む3D座標を算出して固定
- QRを見失った場合はキャラクターを非表示にし、カメラへの追従を防止
- 指定のGLBキャラクター、鍋、家具、食材などのオブジェクトを表示
- QRを地面の高さ・向き・原点として固定し、家具や食材はその場に配置
- QRの印刷面をステージの表側として扱い、裏面表示を防止
- キャラクターだけが地面の高さを保ったまま、ステージ上の調理場所をランダムに巡回
- オーバークック風に「食材を運ぶ → 調理場所へ移動 → 料理する」を繰り返す
- 移動中はGLB内蔵の歩行と手持ち食材、各調理場所では料理アニメーションを自動再生
- キャラクターをタップすると料理を止め、`Idle.fbx` の約4秒の動きを再生してから料理へ復帰
- 効果音のオン・オフ
- `qr.html` で公開URLの印刷用QRコードを作成

## 公開方法

カメラと本格ARの利用には **HTTPSでの公開** が必要です。

1. `ar-character` フォルダを静的ホスティングへアップロードします。
2. `qr-marker.pdf` をA4・100%で印刷します。
3. スマホで印刷したQRを読み取り、ARページを開きます。
4. QR用紙を地面へ置き、AR開始後に同じQRをカメラへ映します。

QR追跡方式のため、固定表示中はQRコードがカメラ内に見えている必要があります。QRを見失った場合は、カメラへの追従を防ぐため表示を消します。

Netlify Drop、Cloudflare Pages、GitHub Pagesなど、静的ファイルをHTTPS配信できるサービスで公開できます。

## ローカル確認

PCで見た目とタップ反応を確認する場合は、このフォルダでWebサーバーを起動します。

```powershell
python -m http.server 4173
```

その後、ブラウザで `http://localhost:4173/` を開きます。PCでは3Dプレビューまたはカメラ合成モードになります。

## 差し替えポイント

- 3Dモデルと料理オブジェクト: `assets/vetnum.glb`
- タップ時の動き: `assets/Idle.fbx`
- セリフ: `app.js` の `messages`
- 画面デザイン: `styles.css`

現在のキャラクターはコードで作った軽量なオリジナルです。GLB形式の専用3Dモデルがあれば、同じ操作仕様のまま差し替えできます。
